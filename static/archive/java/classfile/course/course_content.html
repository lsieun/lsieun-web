主要内容

我们明确了课程主题是.class文件（Java ClassFile），现在就是把整个课程主题拆分成一个个具体的、分阶段讲解的内容。本课程的主要内容就如下所示：

第一章，Basic。这一章主要从“概念”和“实践”两个方面来展开。其中，“概念”部分，我们会介绍一些简单的知识，为后续内容的展开做一个准备；“实践”部分，则主要是下载项目源代码、运行其中的几个类文件、观察运行后的结果，以加深对概念的理解。

第二章（Byte Perspective）、第三章（Attribute Perspective）和第四章（Opcode Perspective），是本课程的核心内容。学了这三章内容，相当于学完了本课程的主体内容。后续章节的内容，都是为了巩固大家对这三章内容的掌握程度。

第二章，Byte Perspective。它从整体上来讲解ClassFile的Structure。这一章内容对应于《The Java Virtual Machine Specification, Java SE 8 Edition》的“Chapter 4. The class File Format”中的从“4.1. The ClassFile Structure”到“4.6. Methods”的内容。

第三章，Attribute Perspective。它是详细的介绍了ClassFile当中的23个具体的Attribute。这一章内容对应于《The Java Virtual Machine Specification, Java SE 8 Edition》的“Chapter 4. The class File Format”中的“4.7. Attributes”部分的内容。

第四章，Opcode Perspective。它是对JVM支持的200多个opcode进行介绍。这一章内容对应于《The Java Virtual Machine Specification, Java SE 8 Edition》的“Chapter 6. The Java Virtual Machine Instruction Set”的内容。

第五章，Java Features Perspective，它是从Java发展的角度来讲解。我们知道，每一个新的Java版本都多多少少的引入了一些新的特性（Feature）。在这部分内容当中，我们把这些新的特性（Feature）放到bytecode的视角去审视，一方面是温习已经学过的知识，另一方面也可能学到一些新的东西。

第六章，Code Examples，它是属于的总结性内容。这一章以总结代码（Code）示例为主。

第七章，The Last，它是最后一章，也是属于的总结性内容。这一章以总结“观念、观点”为主。

第二章（Byte Perspective）、第三章（Attribute Perspective）、第四章（Opcode Perspective）和第五章（Java Features Perspective）都有一个perspective单词，是想表达以不同的“视角”来审视ClassFile。第二章，主要是研究ClassFile的整体结构，而.class文件中存储的是bytecode，第二章在选取名字的时候就选择了bytecode当中的byte部分。第三章，是主要研究ClassFile当中的23个Attribute，因此其名字中就包含了Attribute。第四章，是主要研究200多个opcode，因此其名字中就带有opcode。第五章，主要是从不同版本Java语言特性（Feature）的角度来看ClassFile的内容，因此其名字中带有Feature。

学习曲线

简单来说，

  • 第一章、第二章是属于slow beginning的阶段，主要是打下一些基础。
  • 第三章，有23个具体的Attribute，其难度各易，可能遍布整个学习曲线。
  • 第四章，是属于steep acceleration阶段，是学习突飞猛进的阶段。
  • 第五章，是属于逐步进入plateau阶段，是巩固以前的知识和扩展视野的阶段。
  • 第六章和第七章，是以总结为主，不属于学习曲线的内容，只是为了方便日后查阅。

With repetition of almost any task, learning occurs. A person becomes more efficient or effective at carrying out a task. 笔记：在中文语言环境中，称之为“熟能生巧”。

Progress in skill learning commonly follows an S-shaped curve, with some measure of skill on the Y-axis and number of trials on the X-axis. Progress is slow at first, then a subject may experience a burst of learning that produces a rapid rise on the graph. 笔记：在中文语言环境中，讲“熟能生巧”，但它是一个整体性的结论。而Learning Curve就是表示“熟”和“巧”之间的动态变化关系。

The S-shaped learning curve is most obvious when someone learns a highly complex task. The initial part of the curve rises slowly as a person becomes familiar with basic components of a skill. 笔记：开始的时候，学习进度很慢

The steep ascending phase occurs when there is enough experience with rudiments基本原理 or simple components to start putting it all together. Rapid progress follows until the skill hits a ceiling or stabilizes at a high level. 笔记：中期的时候，进展迅速。

People sometimes refer to plateaus or periods of no improvement during skill learning. However, when experimenters looked for such a phenomenon, they did not find it. 笔记：后期的时候，趋向于稳定。

Fitts and Posner (1967) found gradual improvement with practice in almost all skills. They said flatly there were "no plateaus." Fred Keller of Harvard referred to the "phantom plateau" since one seldom occurred, but people believed that it did. 笔记：有的学者认为，所谓的“瓶颈期”并不存在，是一个人自己幻想出来的事物。

In general, an S-shaped curve of growth levels off达到平衡；趋向平稳 because stability is attained, a resource needed for growth is limited, or a ceiling of performance is reached. 笔记：所谓的“瓶颈期”，其本质上说，是资源不够的问题。也就是说，没有足够的资源让自己技能更提升一步。

Given enough time to practice any skill, one should become very good at it, then improvement stops. This is a plateau of sorts, indicating maximum competence has been attained. 笔记：学习一件新东西，给予足够的时间，一个人就能掌握它，技能也趋于稳定。

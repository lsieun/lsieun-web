generate a Java class

In Oracle JRE 8, the metafactory dynamically generates a Java class using ObjectWeb Asm, which implements a functional interface. Additional fields to the generated class can be added if the lambda expression encloses external variables. This approach is similar to anonymous classes in the Java language — with the following differences:

  • The anonymous class is generated by the Java compiler at compile-time.
  • The class for lambda implementation is generated by the JVM at runtime.

在java.lang.invoke.InnerClassLambdaMetafactory的buildCallSite()会调用spinInnerClass()方法。

@Override CallSite buildCallSite() throws LambdaConversionException { final Class innerClass = spinInnerClass(); // ...... }

对于spinInnerClass()的描述是“Generate a class file which implements the functional interface, define and return the class”。换句话说，这个spinInnerClass()方法会生成一个类，我们想做的事情就是得到这个类的二进制表示形式（byte[]）。

private Class<?> spinInnerClass() throws LambdaConversionException { String[] interfaces; String samIntf = samBase.getName().replace('.', '/'); // ...... // 第一步，生成Class Info部分的信息 cw.visit(CLASSFILE_VERSION, ACC_SUPER + ACC_FINAL + ACC_SYNTHETIC, lambdaClassName, null, JAVA_LANG_OBJECT, interfaces); // 第二步，生成Fields部分的信息 for (int i = 0; i < argDescs.length; i++) { FieldVisitor fv = cw.visitField(ACC_PRIVATE + ACC_FINAL, argNames[i], argDescs[i], null, null); fv.visitEnd(); } // 第三步，（1）生成Methods部分的信息，这里只是生成Constructor generateConstructor(); // ...... // 第三步，（2）生成Methods部分的信息，这里生成其它方法 MethodVisitor mv = cw.visitMethod(ACC_PUBLIC, samMethodName, samMethodType.toMethodDescriptorString(), null, null); mv.visitAnnotation("Ljava/lang/invoke/LambdaForm$Hidden;", true); new ForwardingMethodGenerator(mv).generate(samMethodType); // ...... cw.visitEnd(); // 第四步，将完整的ClassFile表示byte[]的形式 final byte[] classBytes = cw.toByteArray(); // 注意：解决方案一，来自于这里 // If requested, dump out to a file for debugging purposes if (dumper != null) { // 注意：解决方案二，来自于这里 AccessController.doPrivileged(new PrivilegedAction<Void>() { @Override public Void run() { dumper.dumpClass(lambdaClassName, classBytes); return null; } }, null, new FilePermission("<<ALL FILES>>", "read, write"), // createDirectories may need it new PropertyPermission("user.dir", "read")); } return UNSAFE.defineAnonymousClass(targetClass, classBytes, null); }

在上面的代码中，我添加了“第一步”、“第二步”、“第三步”和“第四步”的代码注释，目的是简略性介绍生成类的过程。同时，也标识出了“解决方案一”和“解决方案二”所针对的位置。

解决方案一

首先，找到下面的代码所在的位置，并打上“断点”。

final byte[] classBytes = cw.toByteArray(); // 注意：解决方案一，来自于这里

其次，以Debug方式运行，让代码执行过这一行后暂停。

第三，为classBytes变量添加Evaluate Expression，其内容是Arrays.toString(classBytes)，复制其字符串类型的执行结果。

第四，通过下面代码，将字符串转换成byte[]

public class GoodBoy { public static byte[] getBytes() { String byte_array_str = "[-54, -2, -70, -66, ......]"; String num_str = byte_array_str.replace(" ", "").replace("[", "").replace("]", ""); String[] array = num_str.split(","); int length = array.length; byte[] bytes = new byte[length]; for (int i = 0; i < length; i++) { bytes[i] = (byte) Integer.parseInt(array[i]); } return bytes; } }

最后，利用项目当中的工具类对byte[]内容进行解析。

IMHO: 在有“解决方案二”存在的情况下，为什么要讲“解决方案一”呢？原因是“解决方案二”的存在，相当于jdk提供的一种“福利”，在研究其它代码的时候，未必会有这样的“好事”出现；而“解决方案一”则更通用一些，是通过debug的方式来获取byte[]的内容。

解决方案二

在InnerClassLambdaMetafactory类中，有一个dumper字段，其类型是ProxyClassesDumper。

// For dumping generated classes to disk, for debugging purposes private static final ProxyClassesDumper dumper; static { final String key = "jdk.internal.lambda.dumpProxyClasses"; String path = AccessController.doPrivileged( new GetPropertyAction(key), null, new PropertyPermission(key , "read")); dumper = (null == path) ? null : ProxyClassesDumper.getInstance(path); }

第一种使用方式，添加JVM的启动参数。

java -Djdk.internal.lambda.dumpProxyClasses=/path/to/directory/ HelloWorld

第二种使用方式，在程序代码中设置jdk.internal.lambda.dumpProxyClasses参数。

System.setProperty("jdk.internal.lambda.dumpProxyClasses","D:/git-repo/java8-classfile-tutorial/target/classes");
import java.util.function.Consumer; public class HelloWorld { public static void main(String[] args) { System.setProperty("jdk.internal.lambda.dumpProxyClasses","D:/git-repo/java8-classfile-tutorial/target/classes"); Consumer<String> c = System.out::println; c.accept("Hello"); } }

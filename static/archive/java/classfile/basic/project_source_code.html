项目介绍

常用的功能

常用的功能位于项目的run package下，其功能描述如下：

  • A_File_Hex: 查看一个文件内容的十六进制表示形式。
  • B_ClassFile_Raw: 查看一个.class文件的整体结构。
  • B_ClassFile_Split: 可以将一个.class文件拆分成多个不同的部分。
  • C_Change_MagicNumber: 可以修改一个.class文件的Magic Number。
  • D_Change_CompilerVersion: 可以修改一个.class文件的major version信息。
  • E_ConstantPool: 可以查看一个.class文件的常量池信息。
  • F_ClassInfo: 可以查看一个.class文件中当前类的名字、父类、实现的接口等信息。
  • G_Fields: 可以查看一个.class文件包含的字段信息。
  • H_Methods: 可以查看一个.class文件包含的方法信息。
  • I_Attributes_ClassFile: 可以查看一个.class文件在ClassFile层面的属性信息。
  • I_Attributes_Field: 可以查看一个.class文件在Field层面的属性信息。
  • I_Attributes_Method: 可以查看一个.class文件在Method层面的属性信息。
  • I_Attributes_Code: 可以查看一个.class文件在Code层面的属性信息。
  • K_Code_Locals: 可以查看一个.class文件在特定方法里的Code信息。
  • L_Basic_Stack: 可以查看代码在运行过程中，operand stack和local variables的变化情况。
  • M_ASMStackMapFrame: 它可以修改一个.class文件的内容，方便我们对于特定问题的研究。

常用功能的代码处理逻辑

上述代码中，所遵循的处理思路如下：

  • 第一，提供一个文件的“相对路径”，将“相对路径”转换成“绝对路径”。
  • 第二，通过Java I/O API将“绝对路径”指向的文件的内容读取出来，存放到byte[]中。
  • 第三，对byte[]进行处理。

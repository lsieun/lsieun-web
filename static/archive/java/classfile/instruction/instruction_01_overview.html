Instruction

A Java Virtual Machine instruction consists of a one-byte opcode specifying the operation to be performed, followed by zero or more operands supplying arguments or data that are used by the operation. Many instructions have no operands and consist only of an opcode.

JVM instruction = opcode + operands.

typed instructions

Most of the instructions in the Java Virtual Machine instruction set encode type information about the operations they perform. For instance, the iload instruction loads the contents of a local variable, which must be an int, onto the operand stack. The fload instruction does the same with a float value. The two instructions may have identical implementations, but have distinct opcodes.

For the majority of typed instructions, the instruction type is represented explicitly in the opcode mnemonic by a letter: i for an int operation, l for long, s for short, b for byte, c for char, f for float, d for double, and a for reference. 笔记：对于大部分的些opcode，它们有明确的的type信息。 Some instructions for which the type is unambiguous do not have a type letter in their mnemonic. For instance, arraylength always operates on an object that is an array. 笔记：而有些opcode的type是模糊的。 Some instructions, such as goto, an unconditional control transfer, do not operate on typed operands. 笔记：还有一些opcode不带有type信息。

A few instructions operate on any type. The dup instruction, for example, duplicates the top word of a stack irrespective of its type. Some instructions, such as goto, don't operate on typed values. The majority of the instructions, however, operate on a specific type. The mnemonics for most of these "typed" instructions indicate their type by a single character prefix that starts their mnemonic. A few instructions, such as arraylength or instanceof, don't include a prefix because their type is obvious. The arraylength opcode requires an array reference. The instanceof opcode requires an object reference.

Code Attribute

对于200多个opcode，讲解的顺序，并不是按照“数值大小”的顺序讲解，也不是按照“字母顺序”来讲解，而是从实例代码的角度上去研究它；而实例代码的难度，是逐步提升的，在这个过程当中，我们来学习其中涉及到opcode内容。

从实例代码入手学习。其实，换一种说法，还是要从.class文件开始，这就要求我们熟悉ClassFile和method_info部分的内容。首先，我们需要将.class文件解析成ClassFile结构，然后找到方法对应的method_info结构，再从method_info结构中找到其中的Code属性，最后在Code属性当中找到code[]这部分内容，而code[]这部分内容就包含了opcode的内容。

Code_attribute { u2 attribute_name_index; u4 attribute_length; u2 max_stack; u2 max_locals; u4 code_length; u1 code[code_length]; // 这里是存放opcode的地方 u2 exception_table_length; { u2 start_pc; u2 end_pc; u2 handler_pc; u2 catch_type; } exception_table[exception_table_length]; u2 attributes_count; attribute_info attributes[attributes_count]; }

除了需要了解Code属性之外，也需要了解LocalVariableTable属性，它在我们学习opcode过程当中起着非常重要的作用。

范进中举

最后，是一个“范进中举”的小故事。这个小故事，是我看到一个文章。这个文章，对“范进中举”进行了“新视角”的解读。那么，“旧视角”是指“对科举制度的批判”；而“新视角”是指“人到50多岁了，仍然在追寻自己的梦想”。

范进家徒四壁，穷得连一件像样的衣服都买不起，甚至吃饭都成问题，需要老丈人胡屠夫接济。胡屠夫也瞧不起女婿范进：“我自倒运，把个女儿嫁与你这现世宝，历年以来，不知累了我多少。”

后来，范进想去乡试，因没有盘费，走去同丈人商议，被胡屠户一口啐在脸上，骂了一个狗血喷头，道：“不要失了你的时了！你自己只觉得中了一个相公，就‘癞蛤蟆想吃起天鹅肉’来！我听见人说，就是中相公时，也不是你的文章，还是宗师看见你老，不过意，舍与你的。如今痴心就想中起老爷来！这些中老爷的都是天上的‘文曲星’！你不看见城里张府上那些老爷，都有万贯家私，一个个方面大耳?像你这尖嘴猴腮，也该撒抛尿自己照照！不三不四，就想天鹅屁吃！趁早收了这心，明年在我们行事里替你寻一个馆，每年寻几两银子，养活你那老不死的老娘和你老婆是正经！你问我借盘缠，我一天杀一个猪还赚不得钱把银子，都把与你去丢在水里，叫我一家老小嗑西北风！”一顿夹七夹八，骂的范进摸不着门。

辞了丈人回来，范进心里想：“宗师说我火候已到，自古无场外的举人，如不进去考他一考，如何甘心?”因向几个同案商议，瞒着丈人，到城里乡试。出了场，即便回家。家里已是饿了两三天。被胡屠户知道，又骂了一顿。

如《镜花缘》所言：“盖未进场，如何言中；就如人事未尽，如何言得天命。世上无论何事，若人力未尽，从无坐在家中，就能平空落下随心所欲事来。强求固属不可，至应分当行之事，坐失其机，及至事后委之于命，常人之情，往往如此。”

范进考中之时已经54岁，一直考了二十多次。如果你现在54岁，你还有勇气坚持下来吗？估计很少有这样的人。

在这个小故事当中，我们主要是关注这句话：

宗师说我火候已到，自古无场外的举人，如不进去考他一考，如何甘心?

上面这句话，引用到Java范畴当中来。比如说，我们写了一个.java文件，它实现了非常、非常、非常强大的功能，然后可以通过javac编译成相应的.class文件。那么，现在这个.class文件所面临的处境就是：我有一身的才华（实现各种功能），自古无虚拟机之外的优秀代码，如不进虚拟机当中运行一番，如何甘心?

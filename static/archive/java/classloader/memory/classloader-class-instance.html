ClassLoader + Class + Instance

TOC Toggle Commentary

Instance

  • new关键字：HelloWorld instance = new HelloWorld();

Class

  • Class.forName()方法: Class<?> clazz = Class.forName("sample.HelloWorld");

ClassLoader

  • 方式一：获取当前类的ClassLoader。 clazz.getClassLoader();
  • 方式二：获取当前线程上下文件的ClassLoader。 Thread.currentThread().getContextClassLoader()
  • 方式三：获取系统的ClassLoader。 ClassLoader.getSystemClassLoader()方法: ClassLoader classLoader = ClassLoader.getSystemClassLoader();
  • 方式四：获取调用者的ClassLoader。 DriverManager.getCallerClassLoader()

Class VS. Instance

  • Instance --> Class<?>: Class<?> clazz = instance.getClass();
  • Class<?> --> Instance: Object instance = clazz.newInstance();

ClassLoader VS. Class

  • Class --> ClassLoader: ClassLoader classLoader = clazz.getClassLoader();
  • ClassLoader --> Class: Class<?> clazz = classLoader.loadClass("sample.HelloWorld");

Summing Up

总结：

  • 第一，理解ClassLoader、Class和Instance三者之间的关系。换句话说，通过调用哪个方法，才能建立起一个概念与另一个概念之间的关系。
  • 第二，在理解上述关系的基础上，了解一下在JVM的Runtime Data Area上各自占用的内存区域在哪里。

MethodVisitor

Class Info

第一个要注意的地方，就是MethodVisitor类是一个abstract类。 

public abstract class MethodVisitor {
}

fields

第二个要注意的地方，就是MethodVisitor定义的字段有哪些。

从字段的角度上来说，MethodVisitor定义的字段与ClassVisitor和FieldVisitor有很大的相似性。 

public abstract class MethodVisitor {
    protected final int api;
    protected MethodVisitor mv;
}

Constructor

第三个要注意的地方，就是MethodVisitor定义的构造函数。 

public abstract class MethodVisitor {
    public MethodVisitor(final int api) {
        this(api, null);
    }

    public MethodVisitor(final int api, final MethodVisitor methodVisitor) {
        this.api = api;
        this.mv = methodVisitor;
    }
}

methods

第四个要注意的地方，就是MethodVisitor定义的方法。

在MethodVisitor类当中，定义了许多的visitXxx方法。这些visitXxx方法的调用，也要遵循一定的顺序。

对于visitXxx方法的使用，一方面可以查看Java doc的描述，另一方面也可以参考Java Virtual Machine Specification的Chapter 6. The Java Virtual Machine Instruction Set部分。

    public void visitAttribute(final Attribute attribute);

    public void visitCode();
    public void visitInsn(final int opcode);

    public void visitVarInsn(final int opcode, final int var);
    public void visitTypeInsn(final int opcode, final String type);
    public void visitFieldInsn(final int opcode, final String owner, final String name, final String descriptor);

    public void visitInvokeDynamicInsn(final String name, final String descriptor, final Handle bootstrapMethodHandle,
                                       final Object... bootstrapMethodArguments);
    public void visitJumpInsn(final int opcode, final Label label);
    public void visitLabel(final Label label);
    public void visitLdcInsn(final Object value);
    public void visitIincInsn(final int var, final int increment);
    public void visitTableSwitchInsn(final int min, final int max, final Label dflt, final Label... labels);
    public void visitLookupSwitchInsn(final Label dflt, final int[] keys, final Label[] labels);
    public void visitMultiANewArrayInsn(final String descriptor, final int numDimensions);


    public void visitTryCatchBlock(final Label start, final Label end, final Label handler, final String type);

    public void visitMaxs(final int maxStack, final int maxLocals);
    public void visitEnd();

    // ......
}

These methods must be called in the following order (with some additional constraints specified in the Javadoc of the MethodVisitor class): 

(visitParameter)*
[visitAnnotationDefault]
(visitAnnotation | visitAnnotableParameterCount | visitParameterAnnotation | visitTypeAnnotation | visitAttribute)*
[
    visitCode
    (
        visitFrame |
        visitXxxInsn |
        visitLabel |
        visitInsnAnnotation |
        visitTryCatchBlock |
        visitTryCatchAnnotation |
        visitLocalVariable |
        visitLocalVariableAnnotation |
        visitLineNumber
    )*
    visitMaxs
]
visitEnd.

注意：在对visitXxx方法调用过程中，最后一定要调用visitMaxs()和visitEnd()方法。

联想一下：什么样的人是靠谱的人？所谓靠谱的人，就是凡事有交代，件件有着落，事事有回音！说是三件事，其实是一件事，就是一个人要在协作中完成闭环。所谓闭环，就是一件事有开头，就必有结尾。但是一般人做到这一条并不容易。

对于visitMaxs()和visitEnd()方法的调用，就是明确的声明“这件事情，我做完了”。

Constructor

第三个要注意的地方，就是MethodWriter定义的构造函数。 

final class MethodWriter extends MethodVisitor {
    MethodWriter(final SymbolTable symbolTable, final int access, final String name, final String descriptor,
                 final String signature, final String[] exceptions, final int compute) {
        super(/* latest api = */ Opcodes.ASM9);

        this.symbolTable = symbolTable;
        this.accessFlags = "<init>".equals(name) ? access | Constants.ACC_CONSTRUCTOR : access;
        this.nameIndex = symbolTable.addConstantUtf8(name);
        this.name = name;
        this.descriptorIndex = symbolTable.addConstantUtf8(descriptor);
        this.descriptor = descriptor;
        this.signatureIndex = signature == null ? 0 : symbolTable.addConstantUtf8(signature);
        if (exceptions != null && exceptions.length > 0) {
            numberOfExceptions = exceptions.length;
            this.exceptionIndexTable = new int[numberOfExceptions];
            for (int i = 0; i < numberOfExceptions; ++i) {
                this.exceptionIndexTable[i] = symbolTable.addConstantClass(exceptions[i]).index;
            }
        } else {
            numberOfExceptions = 0;
            this.exceptionIndexTable = null;
        }
        this.compute = compute;
        if (compute != COMPUTE_NOTHING) {
            // Update maxLocals and currentLocals.
            int argumentsSize = Type.getArgumentsAndReturnSizes(descriptor) >> 2;
            if ((access & Opcodes.ACC_STATIC) != 0) {
                --argumentsSize;
            }
            maxLocals = argumentsSize;
            currentLocals = argumentsSize;
            // Create and visit the label for the first basic block.
            firstBasicBlock = new Label();
            visitLabel(firstBasicBlock);
        }
    }
}

    public void visitMaxs(final int maxStack, final int maxLocals) {

            computeMaxStackAndLocal();

            this.maxLocals = maxLocals;

        // ......

这些方法可以分作两个阶段：

  • 第一个阶段，就是对visitXxx()方法调用。对visitXxx()方法调用的过程，就是对MethodWriter中定义的字段赋值的过程。
  • 第二个阶段，就是对computeMethodInfoSize()和putMethodInfo()等方法调用。这些方法，会将MethodWriter定义的字段转换成字节内容（byte[]）。这些字节内容（byte[]）符合method_info的结构。

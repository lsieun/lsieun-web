PKCS#12

Notice that the private key itself doesn’t appear anywhere in the certificate format, nor the CSR format. (This a good thing!) As you can imagine, when dealing with several certificates, it can become difficult to keep track of which private keys correspond to which public keys; some certificates expire, some need to be revoked due to a key compromise, some domains have their own certificates for security purposes, and so on. If you lose track of which private key goes with which certificate, you’re pretty much out of luck; it would be nice to store them together so you can always go back to the source. 笔记：建立public key和private key之间的关系，是非常重要的。

Storing the keys, of course, must be done in a secure way. The private key may be the most sensitive bit of information in the entire system. The PKCS #12 format was designed as a standardized way to transmit any arbitrary bit of data securely — by encrypting it in a standardized way — but in practice it is generally used to store certificates and their corresponding private keys. The PKCS #12 format was standardized from an older, de facto standard named PFX. As such, many applications that generate PKCS #12 files give them the extension .pfx. If you export a certificate and private key from Internet Explorer, for instance, you get a .pfx file. 笔记：PKCS #12可以用来建立certificate和corresponding private key之间的关系。

The PKCS #12 format is actually extremely general — a bit too general, in fact. The top-level structure consists of a version number, a sequence of bit strings, and a MAC over the whole thing. It’s up to the reader of the file to interpret the bit strings to figure out if they’re encrypted and what they contain. 笔记：PKCS #12 format的定义太过宽泛，仍然需要由使用者具体解析其中的内容。

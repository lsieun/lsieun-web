Preimage Attack

In cryptography, a preimage attack on cryptographic hash functions tries to find a message that has a specific hash value. 笔记：Hash算法的正常过程是这样的，“输入数据--> Hash算法 --> 输出数据”；而pre-image attack是逆向进行的，是从“输出数据”入手，希望找到相应的”输入数据“。它的着手点有两处，第一个着手点是”Hash算法“，从Hash算法的计算过程中找出破绽，从而实现逆向计算；第二个着手点是”输入数据“，分析输入数据的本身是不是有某种结构特征，从而发现规律，找到类似的输入数据。

A cryptographic hash function should resist attacks on its preimage (set of possible inputs). 笔记：一个合格的Hash算法，应该具备防御pre-image attack的能力。

In the context of attack, there are two types of preimage resistance:

  • preimage resistance: for essentially all pre-specified outputs, it is computationally infeasible to find any input that hashes to that output, i.e., given y, it is difficult to find an x such that h(x) = y. 笔记：这里是“算法”本身的角度，其关键点是不能进行逆向运算
  • second-preimage resistance: it is computationally infeasible to find any second input which has the same output as that of a specified input, i.e., given x, it is difficult to find a second preimage x′ ≠ x such that h(x) = h(x′). 笔记：这里是“输入数据”的角度，其关键点是找到两个不同的输入数据，让它们有相同的输出结果。

Collision Resistance

These can be compared with a collision resistance, in which it is computationally infeasible to find any two distinct inputs x, x′ that hash to the same output, i.e., such that h(x) = h(x′). 笔记：与pre-image resistance和second pre-image resistance相似的一个概念是collision resistance

The relation between collision resistance, second preimage resistance and preimage resistance is rather subtle, and it depends on the formalization of the definition: Under certain conditions, collision resistance implies second-preimage resistance, but does not guarantee preimage resistance; Conversely, a second-preimage attack implies a collision attack. 笔记：这三个概念的差异很微妙

MD5 Core: (1) Brief Introduction

1. Goal

The goal of MD5 or any secure hashing algorithm, is to reduce an arbitrarily sized input into an n-bit hash. For MD5, n = 128 bits. 笔记：这里从目标的角度来讲MD5，同时也适用于其他的secure hash算法。

2. Input and Output Data

MD5 operates on 512-bit (64-byte) blocks of input. Each block is reduced to a 128-bit (16-byte) hash. 笔记：这里是从输入和输出的角度来讲MD5。

MD5输入数据和输出数据
Hash Input Block Size Output Size
MD5 512-bit (64-byte) 128-bit (16-byte)

If the message to be hashed is greater than 512 bits, each 512-bit block is hashed independently and the hashes are added together, being allowed to overflow, and the result is the final sum. 笔记：这里是从输入的角度来讲MD5。当输入数据大于512-bit时，应该如何处理。

3. Procedure

  • MD5 applies 64 transformations to each input block.
  • The 64 operations themselves consist of 16 repetitions of the four bit flipping functions F, G, H, and I.

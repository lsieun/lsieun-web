SHA256 Core: (1) Brief Introduction

TOC Toggle Commentary

1. Goal

The goal of SHA256 or any secure hashing algorithm, is to reduce an arbitrarily sized input into an n-bit hash. For SHA256, n = 256 bits. 笔记：这里从目标的角度来讲SHA256，同时也适用于其他的secure hash算法。

2. Input and Output Data

SHA256 operates on 512-bit (64-byte) blocks of input. Each block is reduced to a 256-bit (32-byte) hash. 笔记：这里是从输入和输出的角度来讲SHA256。

SHA256输入数据和输出数据
Hash Input Block Size Output Size
SHA1 512-bit (64-byte) 256-bit (32-byte)

3. Procedure

  • 第一步，对输入数据进行padding。输入数据是任意长度的，经过padding之后的数据长度是512-bit (64-byte)的倍数。
  • 第二步，每次取出512-bit (64-byte)的数据，进行block operation操作。
  • 第三步，生成256-bit (32-byte)的输出结果。

4. SHA256 VS SHA1

Everything about SHA-256 is identical to SHA-1 except for the block processing itself and the output length. The block size, padding, and so on are all the same.
